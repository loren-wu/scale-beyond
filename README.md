# 尺度之外 / Scale Beyond

《尺度之外 / Scale Beyond》是一个全屏 3D 宇宙尺度探索网页。它不把网页当作传统的滚动页面，而是把浏览器变成一台安静的宇宙摄影机：用户通过滚轮拉近或拉远尺度，通过拖拽移动视角，从地球表面逐渐退到地月系统、太阳系、银河系，最终进入本星系群。

项目追求真实、克制、深邃的天文纪录片气质，避免赛博霓虹、游戏 HUD、代码雨和过度科幻 UI。核心目标不是堆砌知识点，而是让用户感受到尺度变化本身带来的震撼。

## 当前版本

当前发布候选为 `scale-cosmic6`，已完成七段连续尺度，对应 `scaleValue` 0-1800：

- 全屏 Three.js 场景与 `PerspectiveCamera`
- 滚轮缩放，而不是页面滚动；兼容鼠标滚轮、触控板和不同 `deltaMode`
- 电影级连续转场：统一临界阻尼尺度弹簧、五次连续 easing、分层相机惯性、轻微 FOV 呼吸，以及阶段文案的淡出 / 位移 / 失焦换场
- 精细滚轮手感：微输入死区、残量聚合、反向制动、目标超前限制和按尺度分段的速度预算
- “抓住宇宙”式轨道旋转：右拖时画面跟手向右，旋转与观察中心平移彻底解耦，并带俯仰限制和惯性
- 星球局部层：桌面 5400×2700、移动端 2048×1024 NASA Blue Marble 地貌底图，Black Marble 夜灯、海洋高光、独立云层、大气散射与更清晰的近景采样
- 近实时地球：按当前 UTC 计算太阳直射点和真实晨昏线；NASA GIBS 提供最新可用的每日真彩卫星影像，桌面最高 4096×2048、移动端 1024×512，并在缺测或断网时安全露出 Blue Marble
- 完整星球层：地球整体视角、实时昼夜、高纬极光和独立云层自转；近景降低低分辨率实时图层与云层权重，优先保留高清地貌
- 地月系统层：独立月球、压缩后的地月距离与过渡轨道
- 太阳系层：八大行星、五颗矮行星、主要卫星、精细环带、小行星共振间隙、黄道光、海王星外柯伊伯带、日球层轮廓，以及带颗粒对流、日斑、日珥和双层日冕的太阳；桌面 / 移动端行星程序纹理分别提升至 2048 / 1024 级，并加入多尺度地貌、喷射纹、风暴和涡旋
- 银河天幕：原创可见光银河带以约 24° 倾角弯曲横贯太阳系背景，保留不规则尘埃裂隙和克制的暖核冷盘；桌面仅以低权重、灰度化方式混入 NASA/JPL-Caltech Spitzer GLIMPSE 360 的真实结构
- 恒星邻域层：太阳系淡出、NASA 银河平面观测与外部银河模型连续过渡
- 银河系层：中央棒状核球、每臂独立 phase / pitch 的非对称碎裂旋臂、分叉与羽毛状支脉、尘埃双重衰减、外盘 flare / warp、薄盘 / 厚盘、恒星晕、球状星团和猎户支臂太阳位置标记
- 本星系群层：形态明确不同的银河系、仙女座、三角座和大小麦哲伦云，另有 18 个椭圆、矮球状、不规则、超低表面亮度伴星系，以及单 draw-call 的多形态深空星系场；整体采用更接近纪录片的低饱和暖核冷盘色彩
- 光谱恒星与深空层：恒星按冷蓝、白、暖黄等色温分布，亮星带柔和衍射与轻微闪烁；太阳系外加入克制的反射星云薄雾
- 性能分级：桌面版优先细节，窄屏 compact 版显著降低银河/远景点数和 draw call，并限制像素比
- 纪录片式 UI：七段中英文科学叙事、阶段事实条、距离读数、位置提示、操作提示和七段尺度标记

Earth、Earth at night、clouds、Moon 和 Spitzer GLIMPSE 360 银河平面观测已经随项目本地化，核心七段场景在断网时仍可完整运行。近实时层会额外请求 NASA GIBS；这里的“近实时”是最新可用的每日卫星观测，不是直播摄像或秒级刷新，通常至少滞后一天。本地资源缺失或 GIBS 不可用时会继续使用静态 / 程序化 fallback，避免画面中断。可见光银河天幕是依据 NASA 结构资料约束的原创纹理，不标作 NASA 摄影；行星表面和三维星系结构同样属于体验导向的程序化可视化，不冒充严格等比例天文模拟。素材、署名与结构参考见 [`ASSET_SOURCES.md`](ASSET_SOURCES.md)。

## 交互方式

- 滚轮向后：拉远镜头，进入更宏观尺度
- 滚轮向前：拉近镜头，返回更具体尺度
- 鼠标左键拖拽：抓住宇宙进行轨道旋转；右拖时画面跟手向右
- 移动端：单指轨道旋转、双指缩放
- 键盘：上下方向键或 Page Up / Page Down 切换尺度，左右方向键旋转，Shift + 上下方向键调整俯仰，Home / End 返回两端

## 本地预览

这是一个纯静态网页，不需要构建步骤。Three.js 与 NASA 纹理均已随仓库本地化，预览时不依赖第三方 CDN。可以用任意静态服务器打开项目根目录，例如：

```bash
python -m http.server 4173
```

然后访问：

```text
http://localhost:4173/
```
